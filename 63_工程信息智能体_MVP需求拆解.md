# 63 工程信息智能体 MVP 需求拆解

## 1. 项目目标

在“市政建设第二指挥部”场景下，建设一个可快速上线演示的工程信息智能体后台。系统通过 Mock 数据模拟“文档入库 -> 智能问答 -> 图纸检索 -> 答案溯源”的闭环，重点验证：

- 多类型工程资料可被统一管理
- 用户可通过自然语言提问获得结构化答案
- 答案可追溯到来源文档与页码
- 可进行多轮追问并保持上下文

> 本阶段仅做 MVP 前端与演示级全栈（Next.js + SQLite + Prisma），不接入真实外部知识库与 OCR/向量检索服务。

## 2. 业务对象拆分

### 2.1 核心对象

- 工程项目（Project）
- 工程文档（Document）
- 图纸条目（Drawing）
- 问答记录（QA Session / QA Turn）
- 来源引用（Source Citation）

### 2.2 文档类型

- 设计图纸
- 评估文本
- 政府批文
- 会议纪要
- 合同协议

## 3. 功能需求拆解（MVP）

### 3.1 智能体首页

- 显示项目总览指标：文档总量、图纸总量、今日问答量、高风险事项数
- 显示近期动态时间线：文档入库、问答采纳、图纸调阅
- 显示业务洞察卡片：文档完整率、问答命中率、重点工程进度

### 3.2 知识问答页

- 支持选择“当前工程对象”发起提问
- 支持会话列表、会话切换、新建会话
- 支持多轮追问（保留上下文）
- 每轮回答提供：答案正文 + 来源文档 + 页码 + 可继续追问
- 提供“采纳结果”动作：采纳后写入事件与审计日志（Mock）

### 3.3 文档检索页

- 文档台账列表（名称、类型、项目、状态、更新时间）
- 检索过滤（关键词、文档类型、项目）
- 新增文档（Mock 表单）
- 删除文档（二次确认）
- 状态流转（待解析 -> 解析中 -> 已入库）

### 3.4 图纸查阅页

- 图纸列表（图号、区域、专业、版本、关联文档）
- 关键参数字段（河底标高、桥梁高度、桩号范围等）
- 关键词检索与区域筛选
- 图纸参数提取结果展示（Mock）
- 图纸预览占位区（演示态）与来源信息

### 3.5 管理页（沿用示例骨架）

- 用户管理：新增、编辑、启停用
- 日志审计：按模块/关键词筛选、查看详情
- 系统设置：阈值/开关配置、启停用

## 4. 页面信息架构

- `/dashboard`：智能体首页
- `/qa`：知识问答
- `/documents`：文档检索
- `/drawings`：图纸查阅
- `/assistant`：智能问答工作台（Ant Design X 深度交互）
- `/users`：用户管理
- `/audit-logs`：日志审计
- `/settings`：系统设置

> 根路径 `/` 按示例逻辑重定向到 `/dashboard`，登录体系保持一致。

## 5. 数据字段设计（MVP）

### 5.1 工程主记录（沿用 investModel 表结构映射）

- code：工程编号
- title：工程/问题主题
- modelType：业务类型（知识问答/文档检索/图纸查阅）
- status：流程状态
- riskLevel：风险等级
- owner：责任人
- description：摘要
- sourceType/sourceTitle/sourceBatch：来源追溯

### 5.2 问答关键字段

- project_id（映射为业务对象ID）
- query（提问）
- answer（回答）
- source_doc（来源文档）
- source_page（页码）

## 6. 交互细节约束

- 所有高风险操作必须二次确认
- 所有异步动作必须有 Loading 与成功/失败提示
- 空态必须有业务语义文案，不使用 test1/test2
- 所有数据采用专业 Mock，确保演示可信度

## 7. 非目标范围（本期不做）

- 真实 OCR/向量检索/RAG 服务接入
- 真实文件上传解析流水线
- 权限中心/多租户真实鉴权
- 实时协作与消息推送

## 8. 后续可扩展点

- 将 `generateAiDraft` 对接真实知识检索服务
- 增加文档片段高亮与页内定位
- 增加图纸区域标注与参数自动抽取可信度
- 增加项目级权限与数据隔离
